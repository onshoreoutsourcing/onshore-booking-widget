Booking Widget — Embed Test

Plain HTML page demonstrating the canonical embed snippet. No WordPress, no framework, no build tooling — just a script tag and a target element.

The canonical embed snippet is two lines. Site owners paste this anywhere in their HTML to render the booking form. The data-tenant attribute selects which configured property's bookings to display.

Embed snippet: 
<div id="booking-widget" data-tenant="example-property"></div>
<script src="https://onshorebookings.azurewebsites.net/bookingwidget.js"></script>

Live test below

Update the data-tenant attribute to match a slug configured in the Functions App's BOOKING_TENANTS env var, and update the script src to your Functions App's URL before opening this file:

If the form does not render, check the browser console for errors. Common issues:

  • Widget host URL is wrong or unreachable.
  • data-tenant slug does not match any entry in the BOOKING_TENANTS config.
  • This page's origin is not listed in the tenant's allowedOrigins array.
  • The file is loaded as file://, which has no origin and will fail origin validation. Serve via a local web server (e.g. npx http-server) instead.
  • Widget bundle has not been built / deployed yet.
